# 음성 인터랙션 기반 ADHD 자가진단 지원 시스템 (ADHD 자가진단 ♥)

> **"기존의 지루하고 피로도 높은 텍스트 설문을 넘어, 음성으로 소통하는 메디컬 AI 보조 시스템 프로토타입"**
>
> 본 프로젝트는 브라우저 기반 음성 안내(TTS)와 음성 응답 인식(STT)을 적용하여 사용자의 참여도와 접근성을 극대화한 ADHD 자가진단 보조 도구입니다.

---

## 📅 프로젝트 개요
- **개발 기간**: 2026.04 (약 3주)
- **개발 인원**: 3인 (팀 프로젝트)
- **과정명**: 융합 메디컬 AI with 스마트 웰니스

---

## 🛠️ 기술 스택 및 아키텍처

### 1. Technology Stack
- **Backend**: Python, FastAPI, Uvicorn, Pydantic
- **Frontend**: SvelteKit, Vite, JavaScript
- **Voice Interaction**: Web Speech API (`speechSynthesis` [TTS], `SpeechRecognition` [STT])
- **Data Management**: CSV (유저 및 응답 저장), JSON (임시 데이터)

### 2. System Architecture
본 시스템은 확장성과 유지보수성을 고려하여 3계층 구조(3-Tier Architecture)로 설계되었습니다.

`SvelteKit (화면 출력)` → `FastAPI Routers` → `Services` → `Data (CSV/JSON)`

---

## 🎯 핵심 기능 및 프로세스

1. **사용자 세션 관리**: 안내문 동의 후 고유 유저 ID 발급 및 기존 설문 상태 이어하기 기능 제공
2. **스마트 음성 안내 (TTS)**: 한국어 질문 자동 낭독 및 속도/볼륨 조절 기능 (기본 발화 속도 1.00 최적화)
3. **음성 인식 응답 (STT)**: 사용자의 음성 답변을 텍스트로 변환 후 보기 번호/문구로 매핑
4. **예외 처리 및 Fallback**: 브라우저 미지원 또는 10초 무응답 시 즉시 수동 선택(버튼) 유도 및 TTS/STT 중단
5. **데이터 시각화 및 관리**: 최종 점수 계산(부주의 점수, 과잉행동/충동성 점수), 관리자 페이지 내 성별/연령/지역별 통계 그래프 및 CSV 다운로드 기능

## 👤 나의 기여 및 담당 역할 (My Role)

### 1. 음성 인식(STT) 매핑 문구 범위 확장 및 데이터 분류 고도화
- 사용자가 음성으로 답변할 때 발생할 수 있는 데이터 누락을 방지하기 위해 **인식 범위를 대폭 확장하는 아이디어를 제안하고 로직에 반영**했습니다.
- 단순히 숫자 "1, 2, 3, 4"만 인식하던 기존 방식에서 벗어나, 유저가 자연스럽게 말할 수 있는 "매우 그렇지 않다, 그렇지 않다, 그렇다, 매우 그렇다" 등의 **텍스트 문구까지 정확한 점수로 매핑되도록 범위를 확장**하여 데이터의 유효성과 진단 편의성을 높였습니다.

### 2. 사용자 중심의 예외 처리 및 데이터 휘발 디버깅 (핵심 기여)
- **10초 타이머 기반 Fallback 메커니즘**: 음성 인식 무응답 상황을 방지하기 위해 10초간 응답이 없을 시 시스템 멈춤 대신 수동 선택을 유도하고 진행 중인 TTS/STT 자원을 안전하게 해제(`Clear`)하는 로직을 구현했습니다.
- **설문 완료 사용자 '이어서 하기' 예외 조치**: 이미 최종 설문 작성을 완료하여 캐시(임시 JSON)가 완전히 비워진 사용자가 무심코 '이어서 하기'를 클릭했을 때, 빈 문항(Q20)을 마주하며 기존 데이터가 휘발되는 치명적인 UX 에러를 발견했습니다. 이를 해결하기 위해 사용자의 진행 상태(`status === 'completed'`)를 검증하여, 완료된 유저인 경우 설문 화면을 스킵하고 **즉시 결과창(`/result`)으로 강제 리다이렉트 시키는 복구 안전장치**를 추가했습니다.

### 3. 사용자 식별 고유번호 자동 복사 기능 추가
- 의료 데이터 특성상 철저한 익명성을 위해 임의 생성된 고유 유저 ID를 발급합니다. 이 과정에서 유저가 고유번호를 따로 기록하지 않고 성급하게 설문을 진행했다가 분실하여 처음부터 다시 설문을 수행해야 하는 불편함을 포착했습니다.
- 이를 해소하기 위해 아이디 생성과 동시에 **클립보드에 고유번호가 자동으로 복사되는 편의 로직을 추가**하여 예기치 못한 세션 유실 상황에서도 유저가 쉽게 본인의 이력을 찾을 수 있도록 방어 코드를 구축했습니다.

### 4. 감성적인 사용자 인터페이스(UI) 디자인 기여
- 메디컬 자가진단 서비스 특유의 경직되고 딱딱한 분위기를 완화하고자 메인 페이지 비주얼에 참여했습니다.
- 대문에 따뜻한 하트 이모티콘과 사용자의 긴장을 풀어주는 안성맞춤형 환영 문구들을 배치하여 서비스 접근 친밀도를 높이고 진단 몰입감을 끌어올렸습니다.

## ⚙️ 실행 방법 (Getting Started)

### Backend
```
cd backend
python -m venv .venv
.venv\Scripts\activate # windows
source .venv/bin/activate  # Linux/macOS
pip install -r requirements.txt
uvicorn app.main:app --reload
```

### Frontend
```
cd frontend
npm install
npm run dev
```

### 기본 접속 주소: http://127.0.0.1:5173

## 💡 프로젝트를 통해 얻은 점 (Retrospective)
- 협업 및 버전 관리의 기초 확립: 첫 프로젝트를 수행하며 원격 서버 환경에서 GitHub 리포지토리 연동, 브랜치(Branch) 분리를 통한 원본 백업, 안전한 코드 Push/Pull 등 협업에 필수적인 형상 관리 시스템 흐름을 확실하게 체득했습니다. 또한 백엔드(FastAPI)와 프론트엔드(SvelteKit) 간 API 명세 조율이 프로젝트 성패에 미치는 영향을 깊이 깨달았습니다.
- 비즈니스 로직과 유저 스토리를 연결하는 UX 고민: 단순히 기술을 명세대로 구현하는 수준에 그치지 않고, "유저가 고유번호를 잊어버리면 어떡하지?", "실수로 이어서 하기를 누르면 데이터가 꼬이지 않을까?" 같이 실제 사용자 관점의 예외 시나리오(Edge Case)를 직접 도출하고 해결하는 조치를 취하며 유저 중심 개발자로서 한 단계 성장하는 계기가 되었습니다.
